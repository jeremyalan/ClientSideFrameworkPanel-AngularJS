Pain Points

  • Managing $scope

    Scope Inheritance when binding to primitives
    <div ng-controller="ParentCtrl">

   <-- This text block never updates -->
   <span>(( firstName ))</span>

   <div ng-controller="ChildCtrl">
      <input type="text" ng-model="firstName" />
   </div>
</div>
    Common work-around:
    • Always use a period (.) in ng-model binding expressions
    • Use ControllerAs syntax to leverage "named scopes"
    Check out "Mastering Angular $scope" presented at KCDC 2014
    https://jeremyalan.github.io/MasteringAngularScope

  • Directives API

    • Scope binding in directives (e.g. isolated scopes)
    • Transclusion (wtf?)

  • $digest Cycle

    • $scope changes made asynchronously do not appear
    • Responding to DOM events in directives (click, mouseover, etc.)
    Common work-around:
    • Use Angular services whereever possible. $http vs. $.ajax, $timeout vs. setTimeout, etc.
    • If not possible, call $scope.$apply or $scope.$digest

  • Code Structure

    • Organizing files
    • Breaking application into separate modules
